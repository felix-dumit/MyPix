Pages Included

  • Profile page that user can add profile picture of edit one of the registration fields additionaly more information about the user could be provided
  • Upload page that supports multiple files
  • MyPage that acts as a HUB for user activities
  • Page to manage user images between groups, drag and drop functionality (except IE), user can view slideshow of images in group, add group, delete images, expand and collapse groups
  • References page

Techniques

  • Form Controls

    Used in the registration and login pages, it's done using both html5 required, pattern fields, and javascript event listeners for onchange

  • CSS

    All pages have the same base CSS, and when needed inline styles are applied to some elements. See notes for validation errors.

  • Javascript

    Used in form validation as mentioned above, and more heavily used in the my pictures page, for drag and drop, group creation. Also on upload page.

  • DHTML

    The best examples for this are the upload page, which can create new input fields when the user clicks the button, and most of the functionality of the my pictures page, in which new image groups can be created, images can be moved between groups through drag and drop, groups can be expanded or collapsed, and a trashbin that appears for the user to delete images.
    Also on the login and registration page, small icons indicating if the field is filled correctly or not appear when the user selects a field

    • Notes

    • Logout 'functionality' is merely illustrative
    • There is a search box that would be for users to search other people's images, however no page was implemented for this part of the project
    • When 'uploading' image, icon only shows up if image is from the 'img/user/' folder
    • On picture page, some of the functionality, such as drag and drop between groups doesn't work on Internet Explorer
    • CSS3 validation fails because of selector :valid and :invalid, I don't know any other straight forward ways of doing the same thing, so to avoid overcomplicating things I just left it like that.
    • CSS validation also fails for the file lightbox.css which is included in the lightbox libary, since it is not my code I didn't want to mess with it.
